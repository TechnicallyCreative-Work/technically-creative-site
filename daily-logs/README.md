# Daily Logs — Changelog (v1)

This README tracks changes to the `daily-logs/` folder.  
Follow the conventions below. When we improve the process, we’ll bump to v2, v3, etc.

## Conventions
- **File naming:** `YYYY-MM-DD-<short-label>.json` (e.g., `2025-08-26-voice-log.json`)
- **One entry per file** using your canonical schema (date, timestamp, category, project, status, decisions, risks_blockers, next_steps, summary_bulleted, artifacts_links, tags, notes, ai_summary, webhook_url, execution_mode, header, params, query, body).
- **Commit message:** `chore(logs): <action> for YYYY-MM-DD — <short label>`
